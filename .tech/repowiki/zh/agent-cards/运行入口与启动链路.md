## 什么时候用
用于回答项目从哪里启动、UI/Electron/脚本入口在哪里、改启动配置时应该先看哪些文件。

## 修改入口
- `package.json`: 开发、构建、QA、打包命令和关键依赖来源
- `vite.config.ts`: 开发服务、预览构建和 watcher 忽略目录配置
- `pro-workflow/scripts/commit-validate.js`: 运行入口或高价值文件
- `pro-workflow/scripts/config-watcher.js`: 运行入口或高价值文件
- `pro-workflow/scripts/cwd-changed.js`: 运行入口或高价值文件

## 改代码指南
- 启动失败先拆 Vite、Electron 主进程、native addon、环境变量和 dev bridge。
- 新增入口命令时同步 package.json、脚本、QA 和文档可见入口。

## 验证方式
- npm run build
- npm run qa:knowledge-ui

## 风险点
- 只看到 Vite ready 不代表 Electron 已健康启动。
- native addon 或 appData 路径问题通常要用真实 Electron 验证。

## 检索关键词
