## Functional Detail Map

### FD-01 输入区与任务发起
**对标范围**
- 功能介绍
- CLI支持
- Agent 问答交互
- Output Style

**功能目标**
把一次提问变成一次完整的 Agent 运行配置，而不是只输入一句话。

**入口**
- 主输入框
- 模型下拉
- Agent 下拉
- 输出风格下拉
- 思考模式开关
- 上下文附件区
- 命令触发区

**必须展示的状态**
- 当前 Agent 名称
- 当前模型名称
- 当前输出风格
- 当前思考模式
- 当前执行身份或模式
- 当前任务标题
- 当前上下文附件数量

**细分功能点**
1. 输入框支持自然语言提问。
1. 输入框支持 `/命令` 触发。
1. 输入框支持 `@` 文件上下文插入。
1. 输入框支持本地文件附件。
1. 输入框支持模型快速切换。
1. 输入框支持 Agent 快速切换。
1. 输入框支持输出风格快速切换。
1. 输入框支持思考模式开关。
1. 输入区状态要在发送前可见。
1. 提交后任务标题可自动生成或人工修改。

**交互状态**
- `idle`: 空闲待输入
- `configured`: 已选 Agent / 模型 / 风格 / 模式
- `has_context`: 已挂上下文
- `ready_to_send`: 可提交
- `running`: 正在执行
- `blocked`: 等待外部确认或信息补充

**异常与边界**
- 当附件过多时，输入区必须折叠显示
- 当模型与 Agent 不兼容时，需要显示降级提示
- 当输出风格与当前任务冲突时，应允许快速恢复默认

**验收标准**
1. 用户无需离开输入框即可完成任务配置。
1. 任务状态在发送前是可见且可修改的。
1. 输入区状态切换不会破坏正在编辑的内容。

**我们要集成进来什么**
- 输入区状态徽章
- 模型 / Agent / 风格 / 思考模式四联控件
- 命令触发器
- 上下文 chip 栏

### FD-02 提示词增强
**对标范围**
- 功能介绍

**功能目标**
系统自动给用户输入补齐上下文、约束和输出要求，减少 prompt 工程负担。

**细分功能点**
1. 自动添加任务背景
1. 自动添加输出格式要求
1. 自动添加执行限制
1. 自动添加角色约束
1. 自动添加交付说明
1. 自动添加上下文摘要

**交互要求**
- 增强过程应可感知，但不应打断提交
- 用户应能查看最终送入模型的版本
- 增强内容要能追溯

**异常与边界**
- 若系统无法明确补充内容，应退回原始输入
- 当输入已足够完整时，不应过度扩写

**验收标准**
1. 提交前存在增强逻辑。
1. 用户可查看增强后的内容。
1. 增强不应显著降低输入效率。

### FD-03 文件上下文与附件
**对标范围**
- 功能介绍

**功能目标**
让项目文件、图片、代码、目录快速进入对话上下文。

**入口**
- 附件按钮
- `@` 文件选择
- 最近文件
- 项目目录树

**细分功能点**
1. 添加本地文件附件。
1. 通过 `@` 选取项目内文件。
1. 通过目录树选取文件夹或文件。
1. 支持图片文件引入。
1. 支持代码文件引入。
1. 支持多文件上下文。
1. 支持移除单个上下文项。
1. 支持显示上下文来源。
1. 支持上下文数量上限提示。
1. 支持上下文预览摘要。

**交互状态**
- `empty`
- `adding`
- `ready`
- `overflow`
- `removed`

**异常与边界**
- 图片文件应提示模型是否具备多模态能力
- 过大的文件必须折叠或截断
- 同名文件要展示路径避免歧义

**验收标准**
1. 文件能明确出现在输入区上下文列表中。
1. 文件上下文在任务详情里可回溯。
1. 用户可以随时移除或替换附件。

**我们要集成进来什么**
- 附件 chip
- `@` 文件选择
- 上下文来源展示
- 多文件上下文管理

### FD-04 思考模式
**对标范围**
- 功能介绍

**功能目标**
在速度和质量之间显式切换。

**细分功能点**
1. 思考模式开关
1. 思考模式状态徽标
1. 开启后执行前的规划循环
1. 关闭后直接生成响应
1. 与模型切换联动

**交互状态**
- `on`
- `off`
- `planning`
- `executing`

**异常与边界**
- 开启思考模式不能无限延迟
- 用户应能明确知道当前任务是否进入规划

**验收标准**
1. 用户可单击切换思考模式。
1. 开启时任务具有可感知的规划阶段。
1. 关闭时任务响应路径更短。

### FD-05 Agent 体系
**对标范围**
- Agents

**功能目标**
将不同职责的助手拆成独立可复用的 Agent。

**细分功能点**
1. 展示内置 Agent 列表。
1. 展示用户级 Agent。
1. 展示项目级 Agent。
1. 展示自定义 Agent。
1. 每个 Agent 显示名称、描述、模型、系统提示摘要。
1. 每个 Agent 显示可用工具边界。
1. 支持手动选择 Agent。
1. 支持任务类型触发推荐 Agent。
1. 支持 Agent 间切换。
1. 支持 Agent 独立上下文。

**典型 Agent 类别**
- 调试分析
- 代码审查
- 开发规划
- 故事生成
- UI 草图

**交互状态**
- `visible`
- `selected`
- `recommended`
- `disabled`
- `custom`

**异常与边界**
- 同名 Agent 应按来源区分
- Agent 权限冲突时应提示差异

**验收标准**
1. 用户可查看 Agent 的用途和边界。
1. 用户可切换到不同 Agent。
1. 任务可根据类型命中推荐 Agent。

**我们要集成进来什么**
- Agent 卡片
- 角色选择器
- Agent 配置查看器
- 任务路由推荐
