# 15-核心流程图

## Purpose
用跨模块流程图补足 CLAW 的关键运行链路，让读者不只看到静态容器图，还能快速理解系统如何实际流动。

## Scope
本文件聚焦 4 条主流程：
- 聊天界面代理选择与消息路由
- 任务图拆分与 Worker 合并
- 上下文同步与冲突回写
- 事件到回放与分析生成

## Actors / Owners
- Owner: Architecture + Core Runtime
- Readers: 前端、后端、调度、分析实现者

## Inputs / Outputs
- Inputs: Session、TaskNode、ContextSnapshot、EventEnvelope
- Outputs: 跨模块流程图、实现参考链路

## Core Concepts
- `Chat Agent Picker`: 聊天界面里 Claude Code / Codex 的互斥选择器，默认 `Claude Code`
- `Primary Interactive Agent`: 当前聊天会话的主要交互 Agent
- `Execution Plane`: 负责 Worker 调度、AgentAdapter 和结果回写
- `Evidence Plane`: 负责事件、回放和分析

## Behavior / Flow
### 1. 聊天界面代理选择与消息路由

```mermaid
flowchart LR
    User["User"] --> Picker{"Agent Picker\nDefault = Claude Code"}
    Picker -- "Claude Code" --> ChatCC["Chat Session"]
    Picker -- "Codex" --> ChatCodex["Chat Session"]
    ChatCC --> SessionSvc["Session Service"]
    ChatCodex --> SessionSvc
    SessionSvc --> Registry["Adapter Registry"]
    Registry --> Claude["Claude Code Adapter"]
    Registry --> Codex["Codex Adapter"]
    Claude --> AgentCC["Claude Code"]
    Codex --> AgentCX["Codex"]
```

### 2. 任务图拆分与 Worker 合并

```mermaid
flowchart TD
    Root["Root Task"] --> Split{"Need Split?"}
    Split -- "No" --> Run1["WorkerRun"]
    Split -- "Yes" --> ChildA["Child Task A"]
    Split -- "Yes" --> ChildB["Child Task B"]
    ChildA --> RunA["WorkerRun A"]
    ChildB --> RunB["WorkerRun B"]
    RunA --> Merge["Merge Engine"]
    RunB --> Merge
    Merge --> Parent["Parent Task Update"]
```

### 3. 上下文同步与冲突回写

```mermaid
sequenceDiagram
    participant Hub
    participant Worker
    participant Store as Context Store
    Hub->>Worker: full_sync(ContextSnapshot)
    Worker->>Hub: heartbeat(progress, local state)
    Worker->>Hub: worker_result(ResultSummary, ContextDiff)
    Hub->>Store: load latest snapshot
    Hub->>Hub: detect conflict / merge diff
    alt conflict found
        Hub->>Store: write ConflictRecord
    else merge ok
        Hub->>Store: write new ContextSnapshot
    end
```

### 4. 事件到回放与分析生成

```mermaid
flowchart LR
    Raw["Raw Hooks / UI Actions"] --> Normalize["Event Normalizer"]
    Normalize --> EventStore["Event Store"]
    EventStore --> Timeline["Timeline Builder"]
    Timeline --> Replay["ReplayDocument"]
    Timeline --> Metrics["Metrics Engine"]
    Metrics --> Report["AnalysisReport"]
```

## Interfaces / Types
- 聊天模式只绑定一个 `Primary Interactive Agent`
- 任务图模式允许不同 `TaskNode` 绑定不同 AgentOS
- 所有跨模块链路都必须可回溯到 `EventEnvelope`

## Failure Modes
- 若只有静态架构图没有流程图，实现者会难以理解“消息怎么流、状态怎么变、证据怎么落”
- 若聊天界面支持多 Agent 同时激活，会模糊会话归属和回放边界

## Observability
- 以上 4 条流程都必须有对应事件族支撑回放和诊断
