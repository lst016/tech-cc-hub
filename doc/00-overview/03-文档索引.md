# 03-文档索引

## Purpose
提供 CLAW 1.0.0 文档体系的唯一入口，保证读者能按渐进式披露方式理解系统。

## Scope
本文件定义阅读顺序、波次关系、文档分层和历史输入来源。

## Actors / Owners
- Owner: Architecture
- Readers: 新成员、实现者、评审者

## Inputs / Outputs
- Inputs: 1.0.0 文档全量清单
- Outputs: 阅读路径、实现路径、波次计划

## Core Concepts
- `L0`: 总览层，定义产品边界与术语。
- `L1`: 架构层，定义系统边界、容器与核心组件。
- `L2`: 契约层，定义协议、类型、状态机和持久化。
- `L3`: 运行与演进层，定义 UI、workflow、回放、冲突处理和路线图。

## Failure Modes
- 若新增规范不进入索引，会破坏渐进式披露路径。
- 若在 L0/L1 塞入协议细节，会削弱文档层次感。

## Observability
- Wave 1 完成标志: L0 全部可读、L1 至少 Context/Container 两图成型。
- Wave 2 完成标志: 20/22/24/25 可直接指导实现。
- Wave 3 补强标志: 28/29/36/37 让对象、样例、能力边界和验收口径全部落地。
- 版本化交付开始标志: `40-product/1.0.0` 下已经具备 PRD、FR/NFR、Epic、开发计划与追踪矩阵。
