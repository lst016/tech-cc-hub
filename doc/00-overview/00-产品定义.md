# 00-产品定义

## Purpose
定义 CLAW 的产品定位、边界和 North Star，作为全部后续规范的最高层约束。

## Scope
本文件定义产品是什么、服务谁、解决什么问题，以及 v1 的系统边界。
本文件不定义协议细节、事件 schema、API 字段或 UI 像素级布局。

## Actors / Owners
- Product / Architecture: 陆晟韬
- Primary Readers: 产品、架构、核心实现者
- Secondary Readers: 前端、后端、未来的 AgentOS 适配实现者

## Inputs / Outputs
- Inputs: 既有三份旧草稿、QCon 学习结论、后续 ADR
- Outputs: 产品边界、核心价值、目标用户、v1 非目标、价值主张基线

## Core Concepts
- `CLAW`: 构建在 Claude Code、Codex 等 AgentOS 之上的半托管控制层。
- `AgentOS`: 提供底层推理、工具调用、会话执行和基础权限模型的外部系统。
- `SpecAsset`: workflow、skills、prompts、policies、task templates 等可版本化资产。
- `RuntimeAsset`: session、task、event、snapshot、timeline、replay、analysis 等运行资产。
- `Control Plane`: 用户控制、任务编排、权限决策和协同视图。
- `Execution Plane`: 与 AgentOS 的会话桥接、Worker 调度、事件接入和结果回写。
- `Primary User`: 高频使用 Claude Code / Codex 的单人研发者。
- `Core Value`: 让 Agent 使用过程从一次性执行，进化成可回放、可追责、可持续调优的系统。

## Behavior / Flow
1. 用户通过聊天或任务图定义目标与约束。
2. CLAW 将输入组织为 `SpecAsset + Task Graph`。
3. Hub 将任务分派给 Claude Code、Codex 或未来 AgentOS。
4. 底层执行返回事件、状态和产物，CLAW 统一记录为 `RuntimeAsset`。
5. CLAW 根据事件重建时间线、回放文档和分析报告。
6. 用户根据回放与分析持续调优 spec、workflow 和 skills。

产品主链路：
- 不是“再造一个 Agent”
- 而是“把 Agent 使用过程变成可治理的软件资产”

## Interfaces / Types
- 对用户暴露的一级能力:
  - `Chat Workspace`
  - `Task Graph Workspace`
  - `Replay / Analysis Workspace`
  - `SpecAsset Workspace`
- 对 AgentOS 暴露的一级契约:
  - `AgentAdapter`
  - `EventEnvelope`
  - `Session`
  - `TaskNode`

## Failure Modes
- 若 CLAW 试图重做底层执行内核，会导致项目范围失控。
- 若只做薄壳 GUI，则无法承载 spec/workflow/skills 和可观测闭环。
- 若将 RuntimeAsset 与 SpecAsset 混写，会破坏回放与调优的可追踪性。

## Observability
- 至少要能观测 `用户输入 -> 任务分解 -> AgentOS 执行 -> 事件入流 -> 回放生成 -> 分析输出` 全链路。
- 产品级验收以回放闭环优先，而不是以智能分析花哨程度优先。
