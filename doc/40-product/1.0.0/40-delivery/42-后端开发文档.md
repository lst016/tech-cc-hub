# 42-后端开发文档

## Purpose
定义 1.0.0 后端交付面、服务边界和优先实现顺序，供后端与架构协同拆任务。

## Scope
本文件关注产品交付层面的服务目标、主要对象和验收，不规定内部代码实现方式。

## Behavior / Flow
### Backend Capability Areas

| Area | Must Deliver |
|---|---|
| `Session Service` | 创建、恢复、停止、状态查询 |
| `Agent Adapter Layer` | Claude Code / Codex 统一接入 |
| `Task Service` | TaskNode、依赖、分配、Worker 状态 |
| `Event Service` | 统一事件信封、落盘、订阅 |
| `Replay Service` | 生成 ReplayDocument |
| `Analysis Service` | 生成基础 AnalysisReport |
| `Spec Service` | SpecAsset 持久化、绑定、版本 |
| `Governance Service` | 权限请求、冲突、人工介入记录 |

### Backend Milestones

1. `BE-M1`
   - 跑通 Session + 单 AgentOS + 事件入流
2. `BE-M2`
   - 接入第二个 AgentOS 和统一能力视图
3. `BE-M3`
   - 跑通 Task Graph / WorkerRun / 结果回写
4. `BE-M4`
   - 跑通 Replay / Analysis / SpecAsset
5. `BE-M5`
   - 跑通 Governance 与冲突闭环

### Backend Acceptance

- 事件流可以稳定落盘
- Session / Task / Worker 有可查询状态
- Replay 和 Analysis 可生成
- SpecAsset 可绑定和版本化
- 权限与冲突具有可回放记录

- `Conflict`

## Failure Modes
- 如果后端没有先打通事件主链路，前端工作区很快会失真。
- 如果后端只做存储不做归一化，AgentOS 差异会污染整个产品层。

## Observability
- 后端至少要能统计:
  - 事件写入成功率
  - Replay 生成成功率
  - Analysis 生成成功率
  - 冲突处理闭环率
