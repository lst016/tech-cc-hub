# 41-前端开发文档

## Purpose
定义 1.0.0 前端交付面、页面边界和迭代顺序，供前端实现直接拆任务。

## Scope
本文件关注页面、工作区、核心组件和前端验收，不展开后端协议细节。

## Behavior / Flow
### Screens / Workspaces

| Screen | Must Deliver |
|---|---|
| `Session Sidebar` | Session 列表、当前状态、最近活动 |
| `Chat Workspace` | 消息流、Agent Picker、输入框、停止按钮 |
| `Task Graph Workspace` | 节点视图、依赖、状态、节点控制 |
| `Live Timeline` | 实时事件列表、过滤、跳转 |
| `Replay / Analysis` | 回放入口、分析报告入口 |
| `Spec Assets` | 资产列表、绑定入口、版本历史入口 |
| `Conflicts / Permissions` | 审批与冲突处理面板 |

### Frontend Milestones

1. `FE-M1`
   - 搭建 shell、导航和 Session Sidebar
   - 跑通 Chat Workspace 基础交互
2. `FE-M2`
   - 接入 Live Timeline
   - 增加 Replay / Analysis 入口
3. `FE-M3`
   - 接入 Task Graph 基础视图
   - 支持节点状态和控制动作
4. `FE-M4`
   - 接入 Spec Assets、Permissions、Conflicts

### Frontend Acceptance

- 默认聊天 Agent 为 `Claude Code`
- 聊天界面 Agent 为互斥二选一
- 用户能从聊天跳转到任务、回放、分析和产物
- 用户能看到权限请求和冲突列表
- 主路径在常见桌面窗口宽度下可操作
- 已交付 UI 的用户可见文案默认必须为简体中文
- 品牌名、代码标识、协议字段与原始错误允许保留英文，但不能替代中文主文案
- Electron 客户端 QA 默认按 `window id` 定位真实窗口，不以 `localhost` 页面替代客户端验收

## Interfaces / Types
- `SessionViewModel`
- `AgentPickerViewModel`
- `TaskGraphViewModel`
- `TimelineViewModel`
- `ReplayViewModel`
- `SpecAssetViewModel`

## Failure Modes
- 如果前端只做聊天，不做任务图和证据面板，产品差异会被掩盖。
- 如果前端一次堆太多复杂控件，首版交互质量会失控。

## Observability
- 前端应可上报:
  - `chat_agent_selected`
  - `timeline_filter_changed`
  - `replay_opened`
  - `analysis_opened`
  - `conflict_opened`
