# 48-组件索引

## Purpose

## Behavior / Flow
推荐拆分顺序：
1. 容器组件
2. 交互关键组件
3. 状态 / 展示组件
4. 辅助组件

## Failure Modes
- 如果只拆到页面级，前端实现仍然会在组件边界反复讨论。

## Observability
- 每个组件文档至少定义:
  - 输入状态
  - 输出事件
  - 错误态
