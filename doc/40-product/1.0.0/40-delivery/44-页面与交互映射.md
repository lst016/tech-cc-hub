# 44-页面与交互映射

## Purpose
把 1.0.0 的核心需求映射到具体页面、工作区和关键交互，供前端继续细拆页面任务。

## Scope
本文件只做产品级页面映射，不定义组件实现细节。

## Interfaces / Types
| Surface | Primary Purpose | Related FR | Related Epic |
|---|---|---|---|
| `Session Sidebar` | 管理最近 Session、进入上下文 | `FR-CHAT-002`, `FR-CHAT-005` | `EP-001` |
| `Chat Workspace` | 聊天交互、主 Agent 选择、流式执行 | `FR-CHAT-001`, `FR-CHAT-003`, `FR-CHAT-004` | `EP-001` |
| `Task Graph Workspace` | 任务节点、依赖、状态、节点控制 | `FR-GRAPH-001 ~ 006` | `EP-002` |
| `Live Timeline Panel` | 展示实时事件和过滤 | `FR-EVID-002` | `EP-003` |
| `Replay / Analysis Panel` | 查看回放、报告、比较 | `FR-EVID-003 ~ 005` | `EP-003` |
| `Spec Asset Panel` | 资产列表、绑定、版本 | `FR-SPEC-*` | `EP-004` |
| `Conflict / Permission Panel` | 审批、冲突和人工介入 | `FR-GOV-*` | `EP-005` |

## Behavior / Flow
### Primary UI Flows

1. `Chat Primary Flow`
   - 进入 Session
   - 选择 Agent
   - 发送消息
   - 观察时间线
   - 打开 Replay / Analysis

2. `Task Graph Primary Flow`
   - 创建节点
   - 设置依赖
   - 分配 AgentOS
   - 观察 Worker 状态
   - 处理失败并重试

### UI Gate

- 1.0.0 不要求所有工作区都首发为完整独立页面
- 但必须保证它们在产品中可访问
- 聊天和任务图仍为双主轴，不能降级成附属弹层

- 具体组件拆分见 [48-组件索引.md](./48-%E7%BB%84%E4%BB%B6%E7%B4%A2%E5%BC%95.md)

## Failure Modes
- 如果页面映射不明确，前端实现会再次按“模块名字”而不是“用户路径”拆任务。

## Observability
- 页面级事件至少包含:
  - `sidebar_session_opened`
  - `chat_workspace_focused`
  - `task_graph_opened`
  - `timeline_filtered`
  - `replay_opened`
