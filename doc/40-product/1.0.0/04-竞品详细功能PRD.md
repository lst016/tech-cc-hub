# 04-竞品详细功能PRD

## Purpose
本文件是 `03-竞品功能点PRD` 的细化版，目标不是再讲一遍“竞品有什么”，而是把每个能力拆成可开发、可验收、可映射到我们产品 backlog 的细颗粒度功能定义。

如果说 `03` 是能力地图，那么 `04` 就是功能拆解图。

真正的“一个功能点一个文档”已经继续下沉到 [17-竞品功能拆解索引](./10-requirements/17-%E7%AB%9E%E5%93%81%E5%8A%9F%E8%83%BD%E6%8B%86%E8%A7%A3/00-%E7%AB%9E%E5%93%81%E5%8A%9F%E8%83%BD%E7%B4%A2%E5%BC%95.md)，本文件保留中层拆解和矩阵视角。

## Reading Guide
- 本文件默认把 Z Code 视为“Agent 工作台”而不是“聊天产品”
- 本文件重点放在我们当前要吸收的能力，而不是只记录竞品完整性功能
- 本文件中的 `我们当前状态` 以当前项目主线为准，不代表最终产品上限
- `远程开发` 和 `安全操作确认` 在本轮不作为主线，但会保留为竞品完整性能力

## Functional Detail Map

### FD-01 输入区与任务发起
**对标范围**
- 功能介绍
- CLI支持
- Agent 问答交互
- Output Style

**功能目标**
把一次提问变成一次完整的 Agent 运行配置，而不是只输入一句话。

**入口**
- 主输入框
- 模型下拉
- Agent 下拉
- 输出风格下拉
- 思考模式开关
- 上下文附件区
- 命令触发区

**必须展示的状态**
- 当前 Agent 名称
- 当前模型名称
- 当前输出风格
- 当前思考模式
- 当前执行身份或模式
- 当前任务标题
- 当前上下文附件数量

**细分功能点**
1. 输入框支持自然语言提问。
1. 输入框支持 `/命令` 触发。
1. 输入框支持 `@` 文件上下文插入。
1. 输入框支持本地文件附件。
1. 输入框支持模型快速切换。
1. 输入框支持 Agent 快速切换。
1. 输入框支持输出风格快速切换。
1. 输入框支持思考模式开关。
1. 输入区状态要在发送前可见。
1. 提交后任务标题可自动生成或人工修改。

**交互状态**
- `idle`: 空闲待输入
- `configured`: 已选 Agent / 模型 / 风格 / 模式
- `has_context`: 已挂上下文
- `ready_to_send`: 可提交
- `running`: 正在执行
- `blocked`: 等待外部确认或信息补充

**异常与边界**
- 当附件过多时，输入区必须折叠显示
- 当模型与 Agent 不兼容时，需要显示降级提示
- 当输出风格与当前任务冲突时，应允许快速恢复默认

**验收标准**
1. 用户无需离开输入框即可完成任务配置。
1. 任务状态在发送前是可见且可修改的。
1. 输入区状态切换不会破坏正在编辑的内容。

**我们要集成进来什么**
- 输入区状态徽章
- 模型 / Agent / 风格 / 思考模式四联控件
- 命令触发器
- 上下文 chip 栏

### FD-02 提示词增强
**对标范围**
- 功能介绍

**功能目标**
系统自动给用户输入补齐上下文、约束和输出要求，减少 prompt 工程负担。

**细分功能点**
1. 自动添加任务背景
1. 自动添加输出格式要求
1. 自动添加执行限制
1. 自动添加角色约束
1. 自动添加交付说明
1. 自动添加上下文摘要

**交互要求**
- 增强过程应可感知，但不应打断提交
- 用户应能查看最终送入模型的版本
- 增强内容要能追溯

**异常与边界**
- 若系统无法明确补充内容，应退回原始输入
- 当输入已足够完整时，不应过度扩写

**验收标准**
1. 提交前存在增强逻辑。
1. 用户可查看增强后的内容。
1. 增强不应显著降低输入效率。

### FD-03 文件上下文与附件
**对标范围**
- 功能介绍

**功能目标**
让项目文件、图片、代码、目录快速进入对话上下文。

**入口**
- 附件按钮
- `@` 文件选择
- 最近文件
- 项目目录树

**细分功能点**
1. 添加本地文件附件。
1. 通过 `@` 选取项目内文件。
1. 通过目录树选取文件夹或文件。
1. 支持图片文件引入。
1. 支持代码文件引入。
1. 支持多文件上下文。
1. 支持移除单个上下文项。
1. 支持显示上下文来源。
1. 支持上下文数量上限提示。
1. 支持上下文预览摘要。

**交互状态**
- `empty`
- `adding`
- `ready`
- `overflow`
- `removed`

**异常与边界**
- 图片文件应提示模型是否具备多模态能力
- 过大的文件必须折叠或截断
- 同名文件要展示路径避免歧义

**验收标准**
1. 文件能明确出现在输入区上下文列表中。
1. 文件上下文在任务详情里可回溯。
1. 用户可以随时移除或替换附件。

**我们要集成进来什么**
- 附件 chip
- `@` 文件选择
- 上下文来源展示
- 多文件上下文管理

### FD-04 思考模式
**对标范围**
- 功能介绍

**功能目标**
在速度和质量之间显式切换。

**细分功能点**
1. 思考模式开关
1. 思考模式状态徽标
1. 开启后执行前的规划循环
1. 关闭后直接生成响应
1. 与模型切换联动

**交互状态**
- `on`
- `off`
- `planning`
- `executing`

**异常与边界**
- 开启思考模式不能无限延迟
- 用户应能明确知道当前任务是否进入规划

**验收标准**
1. 用户可单击切换思考模式。
1. 开启时任务具有可感知的规划阶段。
1. 关闭时任务响应路径更短。

### FD-05 Agent 体系
**对标范围**
- Agents

**功能目标**
将不同职责的助手拆成独立可复用的 Agent。

**细分功能点**
1. 展示内置 Agent 列表。
1. 展示用户级 Agent。
1. 展示项目级 Agent。
1. 展示自定义 Agent。
1. 每个 Agent 显示名称、描述、模型、系统提示摘要。
1. 每个 Agent 显示可用工具边界。
1. 支持手动选择 Agent。
1. 支持任务类型触发推荐 Agent。
1. 支持 Agent 间切换。
1. 支持 Agent 独立上下文。

**典型 Agent 类别**
- 调试分析
- 代码审查
- 开发规划
- 故事生成
- UI 草图

**交互状态**
- `visible`
- `selected`
- `recommended`
- `disabled`
- `custom`

**异常与边界**
- 同名 Agent 应按来源区分
- Agent 权限冲突时应提示差异

**验收标准**
1. 用户可查看 Agent 的用途和边界。
1. 用户可切换到不同 Agent。
1. 任务可根据类型命中推荐 Agent。

**我们要集成进来什么**
- Agent 卡片
- 角色选择器
- Agent 配置查看器
- 任务路由推荐

### FD-06 `/Command` 体系
**对标范围**
- / command

**功能目标**
把常用复杂提示词资产化为可调用命令。

**细分功能点**
1. 输入 `/` 后展示命令建议。
1. 支持按名称搜索命令。
1. 支持回车执行命令。
1. 支持命令说明展示。
1. 支持命令提示信息展示。
1. 支持用户级命令自动读取。
1. 支持命令创建。
1. 支持命令编辑。
1. 支持命令删除。
1. 支持命令复用历史。

**命令字段**
- Name
- Prompt
- Description
- Hint

**交互状态**
- `searching`
- `selected`
- `executed`
- `saved`
- `failed`

**异常与边界**
- 以 `/` 开头但无匹配项时要给出空状态
- 长命令提示词要能折叠

**验收标准**
1. `/xxx` 可以直达命令执行。
1. 命令可在设置中维护。
1. 命令内容可长期复用。

**我们要集成进来什么**

- 命令管理页
- 命令模板保存

### FD-11 历史对话编辑
**对标范围**
- 编辑历史对话

**功能目标**
允许用户修改任意历史消息并重新执行，形成新的分支时间线。

**细分功能点**
1. 选择历史消息。
1. 编辑 Prompt 文本。
1. 替换附件。
1. 切换模型。
1. 切换权限模式或执行模式。
1. 重新发送。
1. 回滚后续状态。
1. 清除后续消息与变更。

**交互状态**
- `selected`
- `editing`
- `resubmitting`
- `branched`
- `rolled_back`

**异常与边界**
- 改写历史节点后必须明确提示会影响后续内容
- 回滚必须有明确的恢复点

**验收标准**
1. 历史消息可编辑。
1. 编辑后可从该节点重新分支。
1. 后续消息与文件变更可恢复。

**我们要集成进来什么**
- 历史消息编辑器
- 分支式时间线
- 重新发送
- 节点级回滚

### FD-12 版本管理
**对标范围**
- 版本管理

**功能目标**
把每次 Agent 修改变成可追踪检查点。

**细分功能点**
1. 自动创建检查点。
1. 查看 Review changes。
1. 执行 Undo。
1. 执行 Restore Checkpoint。
1. 通过历史消息定位检查点。
1. 查看多文件 diff。
1. 记录文件变更摘要。

**交互状态**
- `checkpoint_created`
- `reviewing`
- `undo_available`
- `restoring`
- `restored`

**异常与边界**
- 多轮变更必须能合并查看
- 如果没有变更，不应产生空检查点

**验收标准**
1. 每次 Agent 文件修改都有检查点。
1. 用户能看多文件 diff。
1. 用户能撤销最近变更或回到历史节点。

**我们要集成进来什么**
- 检查点
- diff 预览
- undo
- restore
- 版本时间线

### FD-14 工作台工具集
**对标范围**
- 智能体开发环境工具
- 网页浏览器
- 右侧边栏（文件预览区）

**功能目标**
把文件、Git、终端、浏览器、预览和侧栏整合成一个开发工作台。

**细分功能点**
1. 任务管理器。
1. 文件管理器。
1. Git 提交面板。
1. 命令行面板。
1. 网页浏览器。
1. 右侧边栏文件预览。
1. 多标签切换。
1. 布局控制按钮。

**工作台能力**
- 文件树浏览
- 文件预览
- 文件 diff
- 终端执行
- 浏览器预览
- 元素选择
- 视口切换
- 标签切换

**异常与边界**
- 侧栏内容过多时必须可折叠
- 同时打开多个工具时要避免主区被遮蔽

**验收标准**
1. 用户不离开主窗口即可完成多数开发动作。
1. 右侧边栏是复合工作区，不只是信息区。
1. 文件、终端、预览、Git 互相联动。

## Acceptance Summary
如果要把这份竞品拆解真正转成我们的开发计划，最低要做到:
1. 输入区不只是聊天框，而是 Agent 控制台。
1. Command / Skill / Memory / MCP 变成可复用资产层。
1. 历史可以编辑、重发、回滚、分支。
1. 右侧不只是信息展示，而是执行可观测层。
1. 文件 / Git / 终端 / 浏览器构成统一工作台。

## Open Questions
- 我们是否将 `Plugin` 和 `MCP` 合并为一个统一工具市场?
- 我们是否将 `权限模式` 彻底弱化，只保留最轻的状态表达?
- `Output Style` 是否允许用户自定义，还是只保留预设?
- 右栏是否统一承载 `节点详情 / 文件预览 / 终端 / 浏览器 / 执行分析`?
