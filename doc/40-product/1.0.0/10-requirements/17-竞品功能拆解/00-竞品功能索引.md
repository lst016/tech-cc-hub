## Reading Order
1. 先读 `01` 到 `04`，把输入和单次任务控制看清楚。
1. 再读 `05` 到 `10`，把可复用资产和扩展能力看清楚。
1. 接着读 `11` 到 `14`，把历史、版本、可观测和工作台看清楚。
1. 最后看 `15` 到 `19`，补完整性能力。

## Notes
- 每篇文档都是独立功能文档，不依赖总览才能读懂。
- 每篇文档都带流程图，后续可以直接拆给前端或产品设计。
- 当前主线优先级仍然是 `Agent 能力层` 和 `执行可观测层`。
- 除索引页外，每篇文档至少包含 `Goal / Flow / Acceptance`，核心功能还补充了 `Problem / User Story / Scope / State / Edge Cases / Telemetry`。
