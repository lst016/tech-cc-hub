# 15-FR-项目工作区与文件管理

## Purpose
定义工作区、文件产物和本地项目上下文在 1.0.0 中的产品能力。

## Scope
本文件覆盖 workspace 选择、文件树、变更、产物和本地路径语义。

## Actors / Owners
- Owner: Product
- Readers: 前端、后端、架构

## Inputs / Outputs
- Inputs: local-first 原则、文件产物需求
- Outputs: `FR-WS-*`

## Interfaces / Types
`FR-WS-001: MUST - 用户可以为 Session 选择本地工作区`
Acceptance Criteria:
- Session 至少可以绑定一个本地目录
- 绑定的工作区在 UI 中可见
- 工作区变化进入事件流

`FR-WS-002: MUST - 用户可以查看工作区文件树和关键文件变化`
Acceptance Criteria:
- 至少支持目录浏览和文件打开
- 文件变化需可回链到关联任务或事件
- 不要求在 1.0.0 实现完整 IDE 能力

`FR-WS-003: MUST - 系统可以展示任务执行产生的关键产物`
Acceptance Criteria:
- 至少支持文件、回放、分析报告三类产物
- 产物需标明来源 Session / Task
- 打开产物不应打断当前主流程

`FR-WS-004: SHOULD - 用户可以查看工作区中的 diff 或变更摘要`
Acceptance Criteria:
- 至少支持文件列表级变化摘要
- 若底层支持更细粒度 diff，可增强显示
- 变更视图应能从事件或任务跳转进入

`FR-WS-005: MUST - 系统必须保持 local-first 路径语义`
Acceptance Criteria:
- 文档和运行资产默认引用本地路径
- 不因产品层包装而隐藏真实工作区来源
- 若路径失效，系统需明确提示

`FR-WS-006: COULD - 系统可以在产品层提供 Git 同步入口`
Acceptance Criteria:
- 该入口明确属于辅助能力
- 不把 Git 变成运行资产真相来源
- 同步失败不会破坏本地运行资产

## Failure Modes
- 如果工作区只是附属概念，用户很难信任任务结果和产物归属。
- 如果产品层遮蔽真实路径，会损害 local-first 价值。

## Observability
- 必须记录:
  - `workspace_bound`
  - `file_tree_opened`
  - `artifact_opened`
  - `diff_viewed`
