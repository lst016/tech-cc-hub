# 50-需求追踪矩阵

## Purpose
把 `1.0.0` 的目标、需求、Epic 和开发交付建立一条可审计的追踪链。

## Interfaces / Types
| Goal | Requirement Areas | Epics | Story Packs | Component / Controller Layers | Delivery Docs |
|---|---|---|---|---|---|
| `G1` 交互控制成立 | `FR-CHAT`, `FR-WS` | `EP-001` | `36-StoryPack-EP-001` | `48`, `49` | `41`, `44`, `46` |
| `G2` 任务治理成立 | `FR-GRAPH` | `EP-002` | `37-StoryPack-EP-002` | `48`, `49` | `41`, `42`, `44`, `45`, `47` |
| `G3` 证据闭环成立 | `FR-EVID`, `NFR-001~004` | `EP-003` | `TBD` | `TBD` | `42`, `43` |
| `G4` Spec 资产成立 | `FR-SPEC` | `EP-004` | `TBD` | `TBD` | `41`, `42`, `43` |
| `G5` 治理闭环成立 | `FR-GOV`, `NFR-007`, `NFR-012` | `EP-005` | `TBD` | `TBD` | `41`, `42`, `43` |

## Behavior / Flow
使用规则：
1. 新增 FR 或 NFR 时，必须更新本矩阵。
2. 新增 Epic 时，必须明确覆盖哪些 requirement area。
3. 发布验收前，矩阵中不得有 `Goal -> Requirement -> Delivery` 断链。

## Failure Modes
- 若无追踪矩阵，PRD 很快会和开发计划脱节。

## Observability
- 发布前应检查:
  - 是否有未归属的 FR / NFR
  - 是否有未进入 Epic 的 requirement area
  - 是否有未进入交付计划的 Epic
