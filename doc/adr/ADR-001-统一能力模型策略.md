# ADR-001-统一能力模型策略

- Status: Accepted
- Date: 2026-04-19
- Owners: CLAW Core

## Context
CLAW 需要同时接入 Claude Code 和 Codex，但两者的底层执行、事件粒度和扩展能力并不完全同构。若直接暴露差异，上层产品将失去统一心智；若强行完全抽平，则会走向重写 AgentOS。

## Decision
采用“统一优先，扩展保留”的能力模型策略：
- 核心产品流只依赖统一能力模型
- 无法抽平的能力进入 `extension`
- `extension` 不允许成为主流程硬依赖
- 适配器必须声明能力矩阵，并在运行时回填真实支持度

## Consequences
- 好处：
  - 上层产品保持一致体验
  - 未来接入新 AgentOS 时有稳定边界
  - 复杂差异不会污染主契约
- 代价：
  - 适配层复杂度上升
  - 某些高级能力只能在诊断视图中暴露
