# ADR-002-混合双核运行时

- Status: Accepted
- Date: 2026-04-19
- Owners: CLAW Core

## Context
如果只以聚合状态为真相，CLAW 无法生成可信回放；如果只保留事件流而缺少状态快照，复杂会话的查询、恢复和前端呈现成本会很高。

## Decision
采用混合双核运行时：
- `EventEnvelope` 作为证据真相，用于回放、分析和审计
- `Session / Worker / Context` 聚合状态作为产品真相，用于 UI、调度和恢复
- 所有状态变更必须可回链到事件
- 不允许只有状态没有事件，也不允许只有事件没有可查询状态

## Consequences
- 好处：
  - 同时满足回放能力和产品交互性能
  - 便于做时间线、分析报告和状态恢复
- 代价：
  - 存储和实现复杂度提高
  - 必须严格维护事件到状态的映射关系
