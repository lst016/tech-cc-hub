# ADR-003-递归任务图预算与停止策略

- Status: Accepted
- Date: 2026-04-19
- Owners: CLAW Core

## Context
CLAW 允许递归任务图能力，这是产品核心卖点之一。但如果没有预算和停止策略，系统会出现不可控的 Worker 扩散、上下文失控和成本不透明。

## Decision
递归拆分能力在架构上保持开放，但运行时必须受预算和停止策略约束：
- 每个 `TaskNode` 必须绑定 `ExecutionBudget`
- 子任务默认继承并收窄父任务预算与作用域
- Hub 必须记录每次拆分原因和使用的策略
- 当预算耗尽、价值不再提升或用户显式阻止时，必须停止继续拆分

## Consequences
- 好处：
  - 保留了“可无限细拆”的产品表达
  - 避免运行时失控
  - 便于做企业级可控性说明
- 代价：
  - 需要专门的策略治理和更复杂的 UI 提示
