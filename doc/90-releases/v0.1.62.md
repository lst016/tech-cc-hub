# v0.1.62 版本更新说明

## 飞书 IM 稳定性

- 修复旧版实时事件缺少 `sender_type` 时收不到用户消息的问题，同时通过机器人身份和近期出站消息指纹阻止自动回复再次触发机器人。
- 飞书机器人身份暂时不可用时会继续重试发现，不再要求应用 ID 发生变化后才恢复。
- 文本回复改用结构化消息参数，失败时复用幂等键重试，并在可安全确认未送达时回退到纯文本格式。
- 飞书频道总开关与实时聊天开关分别保存，避免切换实时接收时意外关闭整个频道。

## 图片直接投递

- 飞书 IM 会话增加图片投递规则：不能只回复图片 URL。
- 远程图片或工作区外的生成图片会先落到频道工作区，再由回复附件层作为飞书图片消息上传。
- 非飞书频道保持原有提示词和回复行为。

## macOS 发布

- macOS arm64 构建产出的 DMG 和 ZIP 现在会上传到共享 artifact，并进入最终 GitHub Release。
- Windows x64 安装包、更新元数据和启动冒烟检查保持不变。

## 验证

- Electron 频道回归测试 27 项通过、0 失败；1 项 Windows 符号链接用例因权限限制跳过。
- TypeScript、Electron 主进程转译、Vite 生产构建和局部 ESLint 检查通过。
- macOS DMG 的最终产物状态以本标签触发的 GitHub Actions 结果为准。
