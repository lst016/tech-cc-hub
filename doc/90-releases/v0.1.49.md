# v0.1.49 版本更新说明

## 更新内容

- 继承 `v0.1.48` 的启动加速改动：后台预热 Claude Code 子进程、延迟技能管理初始化、减少历史会话列表启动时的同步路径校验。
- 修正远端发布流水线：`release.yml` 恢复为原发布面的 macOS arm64 + Windows x64，不再在 tag 发布时运行 linux/mac x64 验证矩阵。
- Windows 发布继续走 `npm run release:win-x64 -> npm run dist:win -> scripts/package-win-safe.mjs`，并在上传前校验 installer、`latest.yml`、`.blockmap` 和 metadata 文件大小。
- `build.yaml` 保持为手动构建验证入口，不监听 tag、不写 GitHub Release，避免与正式发布链路重复。

## 验证

- `npm run test:electron:build`
- `node --test test/electron/windows-packaging-icon-config.test.mjs`
- `git diff --check`
- `release.yml` / `build.yaml` YAML 解析通过
- 远端 `v0.1.48` 发布 run 已确认失败点为发布矩阵误扩展，`v0.1.49` 用修正后的 workflow 重新按 tag 流程发布。

- `v0.1.48` tag 已存在但 Release 资产未成功发布；不要复用或改写该 tag。
