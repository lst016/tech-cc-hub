# v0.1.45 版本更新说明

## 更新内容

- 输入框工具栏新增 Workflow 按钮，可在单次提示中显式启用 Workflow 模式，按钮状态会在发送后自动复位。
- 合入官方 SDK workflow run center：支持识别 SDK workflow 工具调用、记录 workflow run 状态，并在界面中展示可追踪的 workflow 执行中心。
- Claude Agent SDK 从 `0.3.154` 更新到 `0.3.187`，跟进官方 0.3.x 系列的运行时兼容更新。
- 优化 session continue 的无状态压缩策略：当历史需要应用侧压缩时，避免误用远端 resume 或复用 warm runner 跳过压缩。
- 修复 runner 空成功处理：只有可见 assistant 文本才算有效输出，工具调用空跑会自动重试；runner 无 result 结束时改为明确报错。
- 修复 Windows updater 元数据文件名不一致问题：打包脚本会根据 `latest.yml` 自动生成 installer 和 blockmap 别名，避免更新器指向不存在的文件。

## 验证

- `npm run transpile:electron`
- `npm run test:electron:build`
- `npm run package:win`
- `node --check scripts/package-win-safe.mjs`
- `git diff --check`
- 已验证 `latest.yml` 指向的 `tech-cc-hub-Setup-0.1.45.exe` 和 `.blockmap` 均存在，文件大小与 metadata 匹配。
