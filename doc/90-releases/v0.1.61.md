# v0.1.61 版本更新说明

## 稳定性修复

- 修复 Agent 等待 `AskUserQuestion` 人工回答超过 5 分钟后被 Runner watchdog 误判为空闲并终止的问题。
- 人工确认等待期间暂停 watchdog，回答、取消或会话中止后安全恢复；多个并发确认全部结束前不会提前恢复计时。
- 保留真实 Runner 无事件和长时间无事件的超时保护，避免把正常人工等待误报成运行失败。

## 运行时与工作台

- 完善 BrowserView 导航、渲染内容等待、终端调试、网络/控制台读取和 IPC 就绪状态，提升浏览器工作台在长流程中的可恢复性。
- 增加可复用的可视化协议、文档与产物管理，以及对应的内置技能和 QA 覆盖。
- 完善多线程侧聊、消息去重、回复附件、Lark CLI 通道和工作区会话状态同步。
- 扩展模型目录与模型能力标签，改善多模态、图像生成和运行时路由配置。
- 优化输入框附件、队列消息、剪贴板图片、文件引用和设置页交互。

## macOS 打包

- 修复 macOS CodeGraph runtime 依赖未完整进入应用包的问题。
- `afterPack` 现在会为 macOS arm64 复制 CodeGraph 运行时依赖并验证 OAuth 协议注册。
- 发布流水线继续生成 macOS arm64 安装包，同时保留 Windows x64 安装包、更新元数据和启动检查。

## 验证

- Electron TypeScript 测试编译
- Runner watchdog、AskUserQuestion 和 Runner 生命周期聚焦回归测试
- Electron 主进程转译
- 生产构建
- 修改文件 ESLint 与 diff 检查
- macOS CodeGraph 打包逻辑测试
