# v0.1.60 版本更新说明

## 新功能

- 新增内置 Codex 订阅登录：应用自带官方 Codex 登录运行时，可通过系统浏览器连接 ChatGPT 账号，不再要求用户全局安装 Codex CLI。
- 新增飞书联系人 `@` 提及：输入框可以搜索已授权的飞书联系人，并以结构化提及发送和展示。
- 重做定时任务页面：增加任务概览、搜索、状态筛选、同屏详情、键盘操作、统一删除确认，以及更清晰的创建/编辑和调度摘要。
- 会话中的文件修改改为按用户轮次汇总，并统一展示在该轮底部，避免过程卡片中重复或过早出现修改摘要。
- 图片生成与编辑支持最多 4 张并发结果，部分请求失败时仍保留已成功生成的图片。

## 体验与性能

- 限制后台 BrowserView、已加载历史会话和已完成运行的保留数量，降低长时间使用后的内存占用。
- 优化启动与长会话渲染：避免同步扫描旧消息表，流式文本使用轻量展示，实时 token 估算改为有界采样。
- 大图片附件优先保存文件引用，读取历史记录时移除可由本地文件恢复的旧内联预览，减少 SQLite 会话数据膨胀。
- 从飞书等富文本来源粘贴多段内容时保留各自的 HTTP 链接，同时继续以纯文本方式写入输入框。
- 收起左侧会话栏后完全释放内容宽度，不再保留迷你会话轨道。
- 修复空用户消息被误显示为“已发送附件”，以及生图插件启用时展示消息丢失浏览器上下文的问题。
- 统一应用级弹窗遮罩与 BrowserView 隐藏逻辑，避免设置、确认和反馈弹窗被网页工作台遮挡。

## 稳定性与安全

- Codex OAuth 凭据仅在 Electron 主进程中处理，并通过系统安全存储加密；渲染进程只接收脱敏后的连接状态。
- Codex 上游返回 `401` 时执行一次串行刷新并重试，刷新后的凭据会安全落盘；`403` 权限类响应不会触发重复刷新。
- 单个不可解密的 Codex 凭据不再导致全部 API 配置加载失败，设置页的可选数据源失败也不会清空已有配置。
- 登录错误会清理 URL、Bearer、JWT、授权码和 token 字段，登录完成、取消和应用退出时会回收临时目录与子进程。
- 加强 Windows 打包检查：安装包必须包含 Codex 登录运行时、原生依赖、协议回调配置和正确的应用图标。
