# v0.1.50 版本更新说明

## 更新内容

- 继承启动加速改动：后台预热 Claude Code 子进程、延迟技能管理初始化、减少历史会话列表启动时的同步路径校验。
- 修正远端 Windows 打包脚本：`scripts/package-win-safe.mjs` 在 Windows 下显式使用 `npm.cmd` / `npx.cmd`，避免 GitHub Windows runner 无法通过 `spawnSync(..., shell:false)` 启动 npm/npx。
- 保留修正后的发布流水线：`release.yml` 只运行原发布面的 macOS arm64 + Windows x64，Windows 资产上传前继续校验 installer、`latest.yml`、`.blockmap` 和 metadata 文件大小。

## 验证

- `npm run test:electron:build`
- `node --test test/electron/windows-packaging-icon-config.test.mjs`
- `npm run package:win`
- `git diff --check`
- 远端 `v0.1.49` Windows job 日志已确认失败点为 `package-win-safe.mjs` 启动 `npm run transpile:electron` 前置命令失败。
