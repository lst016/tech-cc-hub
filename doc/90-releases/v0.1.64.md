# v0.1.64 版本更新说明

## 自动更新与 Windows 安装可靠性

- 内网更新源支持自动发现 `vX.Y.Z` 版本目录，并在可信内网不可用时回退到 GitHub Release，减少手工调整更新地址的成本。
- 修复 Windows 自动更新替换旧版本时，开始菜单或桌面快捷方式可能被系统链接跟踪重定向到临时 `old-install` 目录，导致更新完成后无法启动的问题。
- 安装器会在新版文件就位后重建原本存在的快捷方式并恢复应用标识；用户主动删除的快捷方式不会被重新创建。
- 修复 GitHub Release 资产重命名脚本中的正则引号错误，确保标签触发后能够继续校验并发布自动更新产物。
- 保留 Windows 安装包启动冒烟检查、更新元数据、安装包及 blockmap 完整性校验，避免不完整产物进入正式 Release。

## 飞书 CLI 智能体会话

- 飞书用户 `open_id` 现在稳定映射到一个工作区，同一用户的不同 `chat_id` 在该工作区内进入彼此隔离的会话，不再为每个飞书会话重复创建工作区。
- 完善飞书消息卡片的逐轮回复、请求者上下文、心跳、错误回传和终态兜底，让长任务在飞书中的进度与结果更完整。
- 移除已经停用的 `pro-workflow` 内置代码，避免旧工作流继续进入应用包。

## 工作区、插件与运行时

- 增强长会话的分支、后台执行、结果可见性和工作区导航，完成任务不会打断当前操作。
- 完善插件清单、能力授权、运行时隔离和撤权链路；高权限调用默认失败关闭，网络能力和增强界面按明确授权启用。
- 加固 Woo 登录、协作工作流和跨渠道运行状态，减少首次安装或长时间运行后的恢复问题。
- 未受信任的 HTML 继续经过安全渲染边界处理，避免聊天内容逃逸到应用界面。

## 发布产物

- Windows x64 安装包、`latest.yml` 和 blockmap。
- macOS Apple Silicon 与 Intel 的签名、公证（notarization）及自动更新产物。
- GitHub Release 附带统一 SHA-256 校验文件。

## 验证

- Electron 测试 TypeScript 编译通过。
- Windows 安装器快捷方式回归测试与 Codex deep-link 回归测试通过。
- Electron 主进程转译、Vite 生产构建与 Windows NSIS 完整打包通过。
- 已完成真实的 `0.1.62 → 0.1.63 → 0.1.64` 连续自动更新验证：新版自动启动、开始菜单与桌面快捷方式均指向正式安装目录，应用可再次从开始菜单正常启动。
