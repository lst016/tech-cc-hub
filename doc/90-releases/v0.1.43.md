# v0.1.43 版本更新说明

## 更新内容

- 修复聊天输入框 Shift+Enter 换行后光标跳回第一行的问题。
- 移除输入框换行/粘贴路径对 `document.execCommand("insertText")` 的依赖，统一按当前 selection range 写入纯文本草稿并恢复光标位置。
- 补充输入框换行回归测试，锁定换行后继续输入应停留在新行末尾的行为。
- 版本号升级到 `0.1.43`。

## 验证

- `npm run test:electron:build`
- `node --test dist-test\test\electron\prompt-input-native-undo.test.js dist-test\test\electron\prompt-editor-keyboard.test.js dist-test\test\electron\prompt-input-newline-preservation.test.js dist-test\test\electron\prompt-input-clear-on-send.test.js dist-test\test\electron\prompt-input-session-drafts.test.js`
- `npm run build`
- Playwright 实测 `first` + Shift+Enter + `second` 后，输入框 DOM 为 `first<br>second`，焦点仍在输入框，光标 offset 位于第二行末尾。
