# v0.1.31 版本更新说明

## 更新内容

- 修复从 Figma、GitHub 等富文本来源粘贴链接时只保留标题、丢失真实 URL 的问题：当剪贴板 HTML 里包含 `http`/`https` 链接且纯文本不是 URL 时，会自动转成 Markdown 链接。
- 保留普通文本粘贴行为：Postman JSON、纯文本、原始 URL 不会被富文本解析改写，避免再次把 JSON 粘贴撑成富文本布局。
- 将剪贴板纯文本提取逻辑拆到独立工具函数，便于后续继续覆盖更多粘贴来源。
- 版本号升级到 `0.1.31`。

## 验证

- `npm exec -- tsc --project src/electron/tsconfig.json`
- `npm exec -- tsc --project test/electron/tsconfig.json`
- `node --test dist-test\test\electron\clipboard-text.test.js dist-test\test\electron\prompt-input-native-undo.test.js dist-test\test\electron\prompt-input-min-width.test.js dist-test\test\electron\figma-rest-node-index.test.js`
- `npm run package:win`

## 注意

- 当前版本专注修复输入框粘贴路径，没有变更模型、会话、MCP 或发布更新器逻辑。
