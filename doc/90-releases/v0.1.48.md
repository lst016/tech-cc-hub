# v0.1.48 版本更新说明

## 更新内容

- 加快桌面端初始化速度：Claude Code 子进程预热改为后台执行，主窗口不再等待预热完成后才显示。
- 延迟技能管理初始化：启动时只注册技能管理 IPC，首次进入技能相关功能时再初始化技能数据库、默认场景和中央仓库目录。
- 减少历史会话列表的启动开销：会话列表加载不再逐条同步校验历史工作目录真实路径，避免历史路径较多、路径不存在或磁盘较慢时拖慢首屏。
- 收敛发布流水线：`release.yml` 作为唯一 tag 发布入口，Windows 发布改走 `dist:win` 安全打包脚本，`build.yaml` 调整为手动构建验证入口，避免重复发布同一 tag。
- 强化发布资产校验：Release 上传前检查 Windows installer、`latest.yml`、`.blockmap` 和 metadata 文件大小，校验失败不创建 GitHub Release。
