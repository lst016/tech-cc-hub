# 会话时间轴磁吸动效设计

## 目标

让聊天内容左侧的会话时间轴在快速跨节点悬浮时呈现连续、克制的磁吸感，同时保持现有黑灰状态语义、点击跳转和性能边界。参考用户提供的三张 Codex 截图：刻度像一组连续形变的波形，浮卡是同一个对象沿轨道移动，内容在卡片内部自然替换。

## 范围

- 优化刻度展开、浮卡移动和浮卡内容切换三段动效。
- 保持默认态：普通节点为灰色短线，当前问题为黑色短线。
- 保持悬浮态：当前问题始终为黑色，悬浮节点和邻近节点仍为灰色阶梯。
- 保持点击节点切换问题、键盘聚焦、`Escape` 关闭和减少动态效果支持。
- 不增加第三方依赖，不修改聊天卡片、侧栏或消息数据协议。

## 动效结构

### 1. 刻度磁吸波形

每个刻度继续使用固定 `40px` 几何宽度，通过 `transform: scaleX(...)` 表达 `8/10/12/16/24/32/40px` 的视觉长度。所有刻度共享一条约 `300ms` 的轻阻尼弹簧曲线；悬浮目标变化时整组刻度连续从当前比例过渡到下一组比例，不重置动画，也不使用延迟队列。

弹簧曲线使用 CSS `linear(...)` 分段函数，只有约 1% 的末端过冲，避免明显弹跳。若运行时不支持 `linear(...)`，回退到 `cubic-bezier(0.22, 1, 0.36, 1)`。

### 2. 浮卡连续跟随

浮卡在时间轴打开期间始终保持挂载。节点变化只更新浮卡的 Y 方向目标值，不重新播放入场动画。定位继续使用 `translate3d`，并使用约 `320ms` 的轻阻尼弹簧曲线移动；首次出现和最终关闭使用独立的 `160ms` 透明度与轻微 X 位移动画。

这样可以避免当前同一条 transition 同时承担“出现”和“换节点”两种职责，消除机械式急停。

### 3. 内容交叉淡化

浮卡容器移动时，旧内容和新内容短暂共存：

- 旧内容在约 `120ms` 内向上移动 `2px` 并淡出。
- 新内容在约 `160ms` 内从下方 `3px` 淡入。
- 新内容负责卡片正常布局，旧内容作为绝对定位层覆盖其上。
- 快速跨越多个节点时，仅保留最近一层旧内容，不堆积动画队列。

内容切换不改变浮卡外壳的出现状态，因此卡片不会闪烁或重新缩放。

## 状态模型

时间轴保持三个相互独立的状态：

- `activeIndex`：当前滚动位置或点击选择的问题，决定黑色节点。
- `previewTurnIndex`：当前悬浮或聚焦节点，决定刻度目标和浮卡位置。
- `previewContentState`：包含当前内容、上一份内容和递增版本号，驱动交叉淡化。

`openPreview(index)` 更新预览目标和内容状态；如果目标未变化则不产生新动画。`closePreview()` 先关闭浮卡外壳，关闭宽限时间结束后清理预览内容。点击只更新 `activeIndex` 并执行滚动，不复用悬浮状态。

## 组件边界

- `ConversationTurnTimeline.tsx`：拥有交互状态、浮卡外壳和内容双层渲染。
- `conversation-turn-timeline.ts`：保留纯函数，计算刻度宽度、预览偏移和内容切换状态，便于测试快速连续切换。
- UI 样式文件：定义内容淡入、淡出关键帧和减少动态效果覆盖；不引入新的动画库。

## 性能约束

- 禁止动画 `width`、`height`、`top`、`left`。
- 禁止在悬浮路径读取 `offsetTop` 或 `getBoundingClientRect()`。
- 动画属性限于 `transform`、`opacity` 和颜色。
- 24 次连续跨节点采样中，布局次数不得超过 26，P95 帧时间不得超过 `25ms`。
- 不为每一帧写入 React state；浏览器合成层负责插值。

## 可访问性

- `prefers-reduced-motion: reduce` 下禁用弹簧、位移和交叉淡化，直接呈现最终状态。
- 保持节点的 `aria-current`、`aria-describedby`、键盘聚焦和 `Escape` 关闭行为。
- 内容交叉淡化层标记为 `aria-hidden`，辅助技术只读取当前内容一次。

## 测试与验收

1. 单元测试锁定刻度目标、预览偏移和快速连续内容切换的状态结果。
2. 源码守卫禁止重新引入 `width/top` transition、布局读取或动画帧内状态写入。
3. 浏览器 QA 连续跨越至少四个节点，验证浮卡不卸载、内容切换完成、卡片与节点中心对齐。
4. 性能 QA 验证布局次数和 P95 帧时间阈值。
5. 与三张参考截图对比默认态和三个悬浮状态，视觉验收分数不低于 90。
6. 生产构建与现有时间轴、应用壳测试全部通过。

## 非目标

- 不实现真实物理引擎或可配置弹簧面板。
- 不增加 Framer Motion、React Spring 等依赖。
- 不改变浮卡内容字段、节点数量上限或滚动选中算法。
