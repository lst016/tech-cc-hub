# v0.1.21 更新说明

发布日期：2026-05-18
类型：正式 GitHub Release

## 本版重点

- 增强内置 MCP 工具链：浏览器诊断现在能区分可达、重定向、服务错误和 Spring Boot health 503 未就绪状态；端口诊断在 PowerShell 失败时会回退到 netstat，并给出 Windows 下可直接执行的清理命令。
- 扩展 IDEA / Spring Boot 协作能力：IDEA 工具支持按版本或启动器路径筛选，新增 `idea_run` 和 `idea_restart`，可从 MCP 侧启动或重启 Maven / Gradle Spring Boot 项目并返回 PID、日志路径和后续验证提示。
- 稳定 RepoWiki 知识库生成：运行时资源定位兼容开发态和打包态，生成默认走完整目录规划，减少旧 metadata 导致目录过少的问题；嵌入式 Python LLM 客户端改为标准库实现，支持 OpenAI 兼容接口和 Codex OAuth 响应接口。
- 优化用户消息修改流程：用户历史消息可直接原位编辑并重新发送，发送时替换原 user prompt 并裁剪后续历史，避免把“重新修改”请求额外塞回输入框造成上下文污染。
- 改善输入框、Markdown 和运行提示稳定性：标题生成失败会回退到草稿标题，中文输入法 Enter 保护更稳，Mermaid 渲染和源码链接回调减少不必要重渲染；Bash 钩子会保护 Windows 原生命令参数并提醒后台服务还需要端口和健康检查验证。

## 内置工具 / MCP

- `mcp__tech-cc-hub-browser__diagnose_port` 增加 PowerShell encoded command 运行方式，并在失败时自动尝试 `netstat.exe`。
- `mcp__tech-cc-hub-browser__http_ping` 返回 `reachable` 和 `state`，让调用方能区分“服务未启动”和“服务已响应但未 ready”。
- `mcp__tech-cc-hub-idea__idea_status` / `idea_open` 支持 `version` 和 `launcherPath`，多 IDEA 版本机器上不再默认猜最新版本。
- `mcp__tech-cc-hub-idea__idea_run` / `idea_restart` 覆盖 Maven、Gradle、端口清理和 DevTools compile/classes 两类重启路径。

## 知识库 / RepoWiki

- RepoWiki runner 支持 `--force-full` 和 `TECH_CC_HUB_REPOWIKI_FORCE_FULL`，避免复用不完整旧目录。
- Wiki 生成模型选择会避开 embedding 模型字段，优先选择知识库专用模型，再回退到 small / analysis / default / expert 模型。
- sqlite-vec 扩展加载兼容 `app.asar.unpacked` 和 Electron `resourcesPath`，降低打包后向量索引初始化失败概率。
- Python LLM 客户端去除对 litellm 的运行时依赖，减少用户本地 Python 环境差异带来的生成失败。

## 会话 / 输入体验

- 用户消息的“修改”改为原消息内联编辑，保留原附件和浏览器标注、代码引用等上下文块。
- 修改历史消息后，数据库会更新目标 user prompt 并删除其后的消息，新的运行从修订后的上下文继续。
- 输入框会裁剪尾随多余换行，并在空内容或尾部换行时插入稳定哨兵节点，减少 contentEditable 光标异常。
- 会话标题生成超时或失败时会使用本地草稿标题继续创建会话，不再把标题生成失败变成发送失败。
