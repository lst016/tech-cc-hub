# v0.1.20 更新说明

发布日期：2026-05-17
类型：正式 GitHub Release

## 本版重点

- 重做输入框交互：支持富文本式 Slash 命令展示，同时保持底层发送内容仍为原始命令文本。
- 修复输入框 Enter 行为：普通 Enter 发送，Shift+Enter 换行，中文输入法候选确认不会误换行或误发送。
- 增加消息重新修改入口：可从聊天消息或选中文本快速生成“重新修改”请求。
- 补齐输入框和会话历史相关 QA，降低“看起来发出去了但历史回填覆盖消息”的风险。

## 输入框 / Slash 命令

- `/ai-image-gen xxx`、`/agent-reach xxx` 等命令在输入框里显示为带图标的命令 token，视觉上接近独立 Skill/Agent 能力。
- 命令选择后仍保留可编辑参数，实际发送给运行时的内容仍是 `/command argument`，不改变现有 slash command 协议。
- Slash 命令提示支持在光标位置触发，不再只依赖输入开头的 `/`。
- 输入框底部控件收敛为模型、思考强度、Slash 列表、Prompt 优化、发送按钮；发送按钮恢复为黑色主按钮。

## Enter / 中文输入法

- 普通 Enter 会发送当前输入。
- Shift+Enter 会插入换行。
- 中文输入法组合输入期间以及候选确认后的短时间内，Enter 不会触发发送，也不会插入多余换行。
- 增加原生 `beforeinput` 兜底，覆盖 contentEditable 下浏览器绕过 React keydown 的情况。

## 消息重新修改

- 聊天消息支持“重新修改”工作流，可引用整条消息或当前选中文本。
- 生成的修改请求会保留已有输入草稿，并追加明确的修改要求骨架。
- 引用内容以结构化上下文进入输入框，避免用户手动复制长段消息。

## 会话稳定性

- 抽出会话历史 merge 逻辑，避免 running 状态下延迟返回的 history replace 覆盖实时流消息。
- 侧栏会话 hover 菜单提升层级，避免三个点操作被时间文字遮挡。
