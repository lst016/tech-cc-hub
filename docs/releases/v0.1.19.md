# v0.1.19 更新说明

发布日期：2026-05-17
类型：正式 GitHub Release

## 本版重点

- 提升 Knowledge / RepoWiki 链路的稳定性和检索效率。
- 整理 Runner 工具准入逻辑，保留原有拦截行为，但让规则顺序更清楚。
- 撤回 Agent 执行页 v1：该方向没有达到“完整复现聊天记录里的 agent 执行全文”的需求，本版本不包含该 UI。

## Knowledge / RepoWiki

- Knowledge 总览读取关联 workspace 时增加容错：单个异常 workspace 不再导致整个总览失败。
- `knowledge_search` 在跨 workspace 搜索 Agent Cards / RepoWiki 时改为每个 workspace 只打开一次，再按来源聚合结果，减少重复读取。
- workspace 的 Git 绑定信息改为一次 `git rev-parse` 读取仓库状态、HEAD 和分支，降低 overview 刷新时的额外 Git 调用。
- 刷新并收敛 `.tech/repowiki` 生成文档，减少过度展开的旧生成内容，保留更适合当前 Knowledge 工作流的资料结构。

## Runner / 工具准入

- 修复技能环境识别里的中文关键字显示，包括 `飞书` 和 `figma官方`。
- 将工具拒绝规则整理为有序 guard 列表，便于后续维护 blocked shell、PowerShell、SDK Cron、Knowledge index、Figma 路由、Plan 模式和外部浏览技能等拦截逻辑。
